1.1.2. Port and service enumeration

The first reflex is to launch some port and service scan like nmap.

Shell Script
# 
nmap -vvv -sC -sV -A -p- "${ip}" -oG "${file}"

-vvv: gives more verbose

-sC: scan with default NSE scripts

-sV: attempts to determine the version of the service running on port

-A: do an extensive scan on these ports

-p-: scan all the ports

ref/files/tryhackme/pickleRick/nmap_report_10-10-17-158.p.thmlabs.com_2022-10-14T22:25:12.txt

Here we have two service running:

  • ssh on port: 22
  • http on port: 80

At first, I tried to connect to the server via ssh to see if there is the possibility with hydra to brute force the credential. Tough luck! The authentication needs a private key..

So weed need to focus our effort on the service running on port 80 which is our web app.

So I proceed with using gobuster to enumerate hidden files or directories.

1.1.3. Web enumeration

As said, we need to look if there is any hidden page running on the server. Let’s use the dir mode on gobuster and enumerate directories and files.

Shell Script
# 
gobuster dir -q -t 64\
    -x php,sh,txt,cgi,html,css,js,py,jpg,jpeg,png,gif \
    -u "${url}" \
    -w "${directory_wordlist}" \
    -o "${file}"

Let’s break the command down:

  • -q: don’t print the banner and other noise (because I use the -o flag)
  • -t: number of concurrent threads
  • -x: File extension to search fo
  • -u: the target url
  • -w: path to the wordlist
  • -o: output file to write results to

ref/files/tryhackme/pickleRick/gobuster_report_10-10-17-158.p.thmlabs.com_2022-10-14T22:25:12.txt
